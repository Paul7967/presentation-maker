## 1. User flows (сценарии использования)

Ниже описаны пошаговые сценарии с переходами между состояниями экрана. Лимиты и тексты сообщений об ошибках соответствуют [api.md](api.md): макс. 100 слайдов, макс. 20 MB файл, таймаут 120 с.

### 1.1 Успешный сценарий: от ввода до скачивания

| Шаг | Действие пользователя | Состояние экрана | Переход |
|-----|------------------------|------------------|---------|
| 1 | Открывает приложение | **Форма:** пустые поля для слайдов, блок загрузки файла без выбранного файла, кнопка «Сгенерировать» неактивна (или активна, но при отправке будет валидация). | — |
| 2 | Вводит заголовок и пункты первого слайда (при необходимости добавляет слайды) | **Форма:** один или несколько слайдов с полями «Заголовок» и «Пункты»; кнопки «Добавить слайд» / «Удалить слайд» доступны. Подсказка: «Максимум 100 слайдов». | Остаётся форма. |
| 3 | Выбирает файл .pptx в блоке загрузки | **Форма:** отображается имя и размер выбранного файла. Если размер &gt; 20 MB — сразу показывается ошибка «Превышен максимальный размер файла (20 MB)», файл не принимается. | Остаётся форма (с ошибкой при превышении размера). |
| 4 | Нажимает «Сгенерировать» | Валидация на клиенте: есть хотя бы один слайд, файл выбран, слайдов ≤ 100, размер файла ≤ 20 MB. При успехе — переход в состояние загрузки. | **Форма** → **Загрузка**. |
| 5 | Ожидание ответа сервера | **Загрузка:** индикатор (спиннер/прогресс), текст «Генерация…» или аналог; кнопка «Сгенерировать» недоступна; форма при необходимости затемнена или только чтение. | — |
| 6 | Сервер возвращает 200 и файл PPTX | **Успех:** сообщение об успешной генерации, кнопка «Скачать» (и при необходимости опция автоскачивания). Возможность «Создать ещё» возвращает к форме (очищенной или с прежними данными — на усмотрение реализации). | **Загрузка** → **Успех**. |
| 7 | Нажимает «Скачать» (или срабатывает автоскачивание) | Файл сохраняется на устройство. Пользователь может нажать «Создать ещё» → возврат к **Форме**. | **Успех** → **Форма**. |

### 1.2 Ошибки валидации на клиенте (до отправки запроса)

| Ситуация | Действие пользователя | Поведение UI | Состояние |
|----------|------------------------|--------------|-----------|
| Ноль слайдов / пустое описание | Нажимает «Сгенерировать» без ввода слайдов или удалил все слайды | Сообщение об ошибке: «Введите описание хотя бы одного слайда». Блок с ошибкой рядом с блоком слайдов или над формой. Запрос не отправляется. | **Форма** (ошибка). |
| Файл не выбран | Нажимает «Сгенерировать» без выбора файла | Сообщение: «Загрузите образец в формате PPTX». Ошибка у блока загрузки файла. Запрос не отправляется. | **Форма** (ошибка). |
| Файл не .pptx | Выбирает файл с другим расширением | При выборе или при отправке: «Поддерживается только формат PPTX». Файл не принимается или не отправляется. | **Форма** (ошибка). |
| Размер файла &gt; 20 MB | Выбирает файл больше 20 MB | Сразу после выбора или при отправке: «Превышен максимальный размер файла (20 MB)». Файл не принимается. | **Форма** (ошибка). |
| Слайдов &gt; 100 | Добавляет более 100 слайдов и нажимает «Сгенерировать» | Сообщение: «Превышено максимальное количество слайдов (100)». Запрос не отправляется. Подсказка в блоке слайдов: «Максимум 100 слайдов». | **Форма** (ошибка). |

После исправления ввода сообщения об ошибках скрываются (при изменении поля/файла или при повторной попытке отправки — по реализации).

### 1.3 Ошибки ответа API (после отправки запроса)

| Код | Ситуация (пример) | Сообщение в UI (из `detail` или аналог) | Переход | Действие пользователя |
|-----|-------------------|-----------------------------------------|---------|------------------------|
| 400 | Нет template / файл не передан | «Загрузите образец в формате PPTX». | **Загрузка** → **Ошибка**. | Исправить и нажать «Повторить». |
| 400 | Нет/пустое поле slides | «Передайте описание слайдов в поле slides». | **Загрузка** → **Ошибка**. | Исправить и «Повторить». |
| 400 | Неверный JSON в slides | «Некорректный формат описания слайдов». | **Загрузка** → **Ошибка**. | Исправить и «Повторить». |
| 400 | 0 слайдов в описании | «Введите описание хотя бы одного слайда». | **Загрузка** → **Ошибка**. | Добавить слайд и «Повторить». |
| 400 | Слайдов &gt; 100 | «Превышено максимальное количество слайдов (100)». | **Загрузка** → **Ошибка**. | Уменьшить число слайдов и «Повторить». |
| 400 | Нарушение лимитов (длина заголовка/пункта, кол-во пунктов) | «Превышен лимит: &lt;конкретика&gt;» (из `detail`). | **Загрузка** → **Ошибка**. | Исправить и «Повторить». |
| 400 | Файл не PPTX | «Поддерживается только формат PPTX». | **Загрузка** → **Ошибка**. | Загрузить другой файл и «Повторить». |
| 413 | Размер файла &gt; 20 MB | «Превышен максимальный размер файла (20 MB)». | **Загрузка** → **Ошибка**. | Загрузить файл меньшего размера и «Повторить». |
| 422 | Ошибка валидации полей (Pydantic и т.п.) | Текст из `detail` (объединённые `msg` или строка). | **Загрузка** → **Ошибка**. | Исправить данные и «Повторить». |
| 500 | Файл повреждён / не читается | «Файл образца повреждён или не может быть прочитан». | **Загрузка** → **Ошибка**. | Загрузить другой файл и «Повторить». |
| 500 | В образце нет слайдов | «В файле образца не найден ни один слайд». | **Загрузка** → **Ошибка**. | Загрузить другой файл и «Повторить». |
| 500 | Сбой генерации | «Не удалось сгенерировать презентацию». | **Загрузка** → **Ошибка**. | «Повторить» или изменить данные. |
| 504 | Таймаут (&gt; 120 с) | «Превышено время ожидания. Попробуйте уменьшить количество слайдов или размер образца». | **Загрузка** → **Ошибка**. | Уменьшить объём и «Повторить». |

Во всех случаях состояния **Ошибка**: отображается сообщение, кнопка «Повторить» (возврат к отправке формы с текущими данными) и возможность редактировать форму и затем снова нажать «Сгенерировать». После «Повторить» или «Сгенерировать» переход: **Ошибка** → **Загрузка** → **Успех** или **Ошибка**.

### 1.4 Диаграмма переходов состояний экрана

```
[Форма] ──(Заполнено, «Сгенерировать»)──► [Загрузка]
   ▲                                          │
   │                                          ├──(200 + файл)──► [Успех] ──(«Скачать»)──► (файл сохранён)
   │                                          │                        │
   │                                          │                        └──(«Создать ещё»)──► [Форма]
   │                                          │
   │                                          └──(4xx/5xx)──► [Ошибка]
   │                                                               │
   └──────────────────(«Повторить» / редактирование + «Сгенерировать»)──┘
```

## 2. Структура экранов и wireframes (текстовое описание)

Однозначное описание экранов для разработчика и тестировщика. Графические макеты не приводятся.

### 2.1 Общая компоновка страницы

- **Верхняя часть:** заголовок приложения (например, «Presentation Maker» или логотип + название).
- **Основная область:** контент в зависимости от состояния (форма / загрузка / успех / ошибка).
- **Подвал (опционально):** минимальная информация (версия, ссылка на справку) — по решению команды.

Вертикальная прокрутка допустима при большом количестве слайдов. Ширина контента ограничена (см. [ui-spec.md](ui-spec.md)) для читаемости.

---

### 2.2 Экран: Форма ввода (основное состояние)

**Назначение:** ввод описания слайдов и загрузка образца PPTX.

**Блоки сверху вниз:**

1. **Блок «Описание слайдов»**
   - Заголовок блока (например, «Слайды»).
   - Подсказка: «Добавьте один и более слайдов. Максимум 100 слайдов. Заголовок — до 500 символов, пункт — до 2000 символов, до 50 пунктов на слайд.» (значения из [api.md](api.md) — при необходимости сократить до «Максимум 100 слайдов» в подсказке).
   - Список карточек/секций слайдов. Каждый слайд:
     - Номер или метка слайда (например, «Слайд 1», «Слайд 2»).
     - Поле ввода «Заголовок» (одна строка или textarea — по стилю).
     - Поле ввода «Пункты»: многострочное поле или список полей (каждый пункт — строка); возможность добавлять/удалять пункты. Для MVP — один уровень пунктов.
     - Кнопка «Удалить слайд» (для каждого слайда, кроме случая одного слайда — тогда либо скрыть, либо оставить с подтверждением, чтобы не остаться без слайдов).
   - Кнопка «Добавить слайд» под списком слайдов.
   - Область отображения ошибок валидации слайдов: при 0 слайдов или слайдов &gt; 100 — сообщение рядом с этим блоком (см. п. 1.2).

2. **Блок «Образец презентации»**
   - Заголовок блока (например, «Файл-образец (PPTX)»).
   - Подсказка: «Выберите файл .pptx до 20 MB. Стиль первого слайда будет применён ко всей презентации».
   - Контрол выбора файла (input type file с accept=".pptx" или соответствующий MIME). После выбора — отображение имени файла и размера (например, «presentation.pptx, 2.5 MB»).
   - При ошибке (не тот формат, размер &gt; 20 MB) — сообщение об ошибке под/рядом с контролом (тексты из п. 1.2).

3. **Кнопка отправки**
   - Основная кнопка «Сгенерировать».
   - Рекомендуется: активна только при наличии хотя бы одного слайда и выбранного файла в пределах лимитов; иначе — неактивна или при нажатии показываются ошибки валидации (согласно US-3).

**Состояние полей:** все поля редактируемые; при повторном заходе после ошибки значения можно сохранять (форма не сбрасывается до явного «Создать ещё» или перезагрузки страницы — по реализации).

---

### 2.3 Экран: Загрузка (ожидание ответа)

**Назначение:** информировать пользователя о том, что запрос выполняется.

- Форма может оставаться видимой в режиме только чтения или быть скрыта/свёрнута (по решению).
- По центру или в области основного контента:
  - Индикатор загрузки (спиннер или скелетон).
  - Текст: «Генерация презентации…» или «Подождите, идёт создание файла».
- Кнопка «Сгенерировать» недоступна; повторная отправка до завершения запроса блокируется (US-3).
- Опционально: информация о таймауте («Максимальное время ожидания — 2 минуты») — по желанию.

---

### 2.4 Экран: Успех

**Назначение:** сообщить об успешной генерации и дать возможность скачать файл.

- Краткое сообщение: «Презентация успешно создана» (или аналог).
- Кнопка «Скачать» — по нажатию скачивается полученный файл (имя по умолчанию, например `presentation.pptx`, или из заголовка ответа `Content-Disposition`).
- Опционально: автоскачивание при получении 200 (тогда кнопка «Скачать» — для повторного скачивания).
- Кнопка «Создать ещё» (вторичная): возврат к форме (очищенной или с прежними данными — см. реализацию). Переход в состояние **Форма**.

---

### 2.5 Экран: Ошибка

**Назначение:** показать причину сбоя и дать возможность повторить попытку.

- Блок сообщения об ошибке (визуально выделен, см. [ui-spec.md](ui-spec.md)): текст из ответа API (`detail`) или сформированный из `detail` для 422. Без технического жаргона.
- Кнопка «Повторить»: повторная отправка текущих данных формы (переход **Ошибка** → **Загрузка**).
- Форма с введёнными данными остаётся доступной для редактирования; пользователь может изменить данные и нажать «Сгенерировать» (тогда тоже переход в **Загрузка**).
- При необходимости — кнопка «Изменить данные» или просто прокрутка к форме, если форма выше по странице.

---

### 2.6 Сводка состояний и элементов

| Состояние | Видимые блоки | Ключевые элементы |
|-----------|----------------|-------------------|
| **Форма** | Описание слайдов, образец PPTX, кнопка «Сгенерировать» | Поля заголовка/пунктов, добавление/удаление слайда, выбор файла, имя/размер файла, ошибки валидации |
| **Загрузка** | Индикатор, текст ожидания | Кнопка «Сгенерировать» недоступна |
| **Успех** | Сообщение об успехе, кнопки | «Скачать», «Создать ещё» |
| **Ошибка** | Сообщение об ошибке, форма (редактируемая) | Текст ошибки, «Повторить», «Сгенерировать» после правок |
