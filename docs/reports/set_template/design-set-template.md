**Связь:** [Требования](requirements-set-template.md), [Архитектура](architecture-set-template.md), [План внедрения](../../plans/dev_plan_set_template.md)

## 1. Режим «По слайдам»

### 1.1. Размещение в SlideCard

В карточке слайда (SlideCard) элементы располагаются в порядке:

1. **Заголовок** (существующее поле)
2. **Пункты** (существующий блок)
3. **Тип шаблона (layoutId)** — новое поле
4. **Заметки докладчика (notes)** — новое поле

Стиль и отступы согласованы с текущим видом карточки (form-group, подписи, подсказки по лимитам).

### 1.2. Поле «Тип шаблона»

- **Вид:** поле ввода (input type="text" или number — по решению разработчика) или выпадающий список (select).
- **Поведение до загрузки образца:** список layout’ов из образца неизвестен. Рекомендуется **универсальное поле ввода**: пользователь может ввести число (индекс layout, 0-based) или имя layout (строка). Подсказка рядом с полем: «Индекс (0, 1, …) или имя layout из образца. Оставьте пустым для первого шаблона».
- **Поведение после загрузки образца (опционально):** при наличии API/данных со списком layout’ов можно заменить поле на выпадающий список с вариантами «Индекс: 0», «Индекс: 1», «Title and Content» и т.д. В первой итерации достаточно одного поля ввода (число или строка).
- **Значение по умолчанию:** пустое (undefined/отсутствует) или 0 — при отправке на API интерпретируется как «первый layout».
- **Доступность (a11y):** label с явной связью (htmlFor/id), при необходимости aria-describedby для подсказки.

### 1.3. Поле «Заметки докладчика»

- **Вид:** textarea (многострочное поле), 2–4 строки по высоте (rows), возможность расширения при вводе — по усмотрению.
- **Placeholder:** «Заметки докладчика для этого слайда (необязательно)».
- **Подсказка/счётчик:** отображать «N / 4000» символов (как для заголовка), при достижении лимита — визуально выделить или показать предупреждение.
- **Валидация на клиенте:** при превышении 4000 символов — сообщение об ошибке и/или блокировка отправки; при переключении в «Через JSON» — та же проверка в parseAndValidateSlides.
- **Доступность:** label «Заметки докладчика», aria-describedby для подсказки и счётчика, при ошибке — aria-invalid и связь с сообщением об ошибке.

### 1.4. Поведение до/после загрузки образца

- **До загрузки образца:** поле «Тип шаблона» доступно как универсальный ввод (индекс или имя). Подсказка: «После загрузки PPTX-образца можно указать индекс (0, 1, …) или имя layout».
- **После загрузки образца:** без изменения поведения (универсальное поле); при появлении в будущем эндпоинта со списком layout’ов — можно показывать select с вариантами.

## 2. Режим «Через JSON»

### 2.1. Формат в подсказке и примере

- В подсказке под textarea указать расширенный формат: «Поля объекта слайда: "title", "items", опционально "layoutId" (число — индекс layout, или строка — имя), "notes" (строка, до 4000 символов).»
- Пример в placeholder или в form-hint:
  `[{"title": "Титул", "items": ["Пункт 1"], "layoutId": 0, "notes": "Заметки"}]`
- Обновить существующую подсказку про лимиты: добавить «заметки — до 4000 символов».

### 2.2. Ошибки валидации новых полей

- При парсинге/валидации (parseAndValidateSlides): если `notes` не строка — ошибка «Некорректный формат: поле "notes" должно быть строкой».
- Если длина `notes` превышает 4000 — «Превышен лимит: заметки слайда N — не более 4000 символов».
- Поле `layoutId`: если присутствует и не число и не строка — «Некорректный формат: поле "layoutId" должно быть числом или строкой». Ошибки показывать в том же блоке (json-error), что и остальные ошибки валидации JSON.

---

## 3. Согласованность и доступность (a11y)

- **Единый стиль:** новые поля используют те же классы (form-group, label, подсказки), что и заголовок/пункты в SlideCard.
- **Фокус и порядок табуляции:** после «Пункты» и кнопки «Добавить пункт» — фокус переходит на «Тип шаблона», затем «Заметки докладчика».
- **Связь label и поля:** у каждого поля уникальный id (например, `slide-layout-${index}`, `slide-notes-${index}`) и соответствующий htmlFor у label.
- **Ошибки:** при ошибке валидации notes — роль alert, aria-live при необходимости для динамических сообщений.

## 4. Сводка для разработчика и тестировщика

| Элемент | Режим «По слайдам» | Режим «Через JSON» |
|--------|---------------------|---------------------|
| layoutId | Поле ввода (число или строка) в SlideCard, подсказка про индекс/имя | Поле в объекте слайда; валидация типа (number/string) |
| notes | Textarea в SlideCard, лимит 4000, счётчик символов | Поле в объекте слайда; валидация длины ≤ 4000 |
| До загрузки образца | Поле layoutId доступно, подсказка про индекс/имя | — |
| Переключение режимов | При сериализации включать layoutId и notes; при парсинге сохранять в state | При парсинге проверять layoutId и notes, писать в state |
