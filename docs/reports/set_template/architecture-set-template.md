## 1. Формат данных слайда

### 1.1. Расширенная структура

Формат слайда в API и JSON:

```json
{
  "title": "Заголовок слайда",
  "items": ["Пункт 1", "Пункт 2"],
  "layoutId": 0,
  "notes": "Текст заметок докладчика."
}
```

- **title** (обязательно) — строка, заголовок слайда.
- **items** (обязательно) — массив строк, пункты списка.
- **layoutId** (опционально) — число (индекс layout) или строка (имя layout). При отсутствии или невалидном значении используется первый доступный layout образца.
- **notes** (опционально) — строка, заметки докладчика. Пустая строка допускается. Максимальная длина — см. раздел 3.

### 1.2. Тип и допустимые значения layoutId

- **Целое число (int):** интерпретируется как **индекс** layout в коллекции `prs.slide_layouts` (0-based). В python-pptx доступ: `prs.slide_layouts[idx]`.
- **Строка (str):** интерпретируется как **имя** layout. В python-pptx 1.x: `prs.slide_layouts.get_by_name(name)` возвращает `SlideLayout` или `None`.
- При невалидном значении (индекс вне диапазона, имя не найдено) — **fallback на первый layout** образца (`prs.slide_layouts[0]`), без ошибки валидации.

## 2. Стратегия маппинга layoutId → layout образца

1. Открыть образец PPTX через python-pptx; коллекция layout’ов: `prs.slide_layouts` (принадлежит первому slide master).
2. Для каждого слайда при генерации:
   - Если передан `layoutId`:
     - если тип **int**: при `0 <= layoutId < len(prs.slide_layouts)` использовать `prs.slide_layouts[layoutId]`, иначе fallback на `prs.slide_layouts[0]`;
     - если тип **str**: вызвать `prs.slide_layouts.get_by_name(layoutId)`; если результат не `None` — использовать его, иначе fallback на `prs.slide_layouts[0]`.
   - Если `layoutId` отсутствует (или не передан) — использовать `prs.slide_layouts[0]`.
3. Создание слайда: `prs.slides.add_slide(chosen_layout)`.
4. Извлечение списка layout’ов для UI (опционально): при необходимости эндпоинт или вспомогательная функция может возвращать список `[{"index": i, "name": layout.name} for i, layout in enumerate(prs.slide_layouts)]` после парсинга образца. В первой итерации можно обойтись без отдельного API: в режиме «По слайдам» — универсальное поле ввода (число или строка), в «Через JSON» — описание формата в подсказке.

## 3. Лимиты

| Параметр | Значение | Примечание |
|----------|----------|------------|
| Максимальная длина поля **notes** на один слайд | **4000 символов** | По аналогии с пунктами (2000) — удвоено, т.к. заметки могут быть объёмнее одного пункта. Валидация на бекенде и фронтенде. |

- Пустая строка `notes: ""` и отсутствие поля `notes` не требуют ошибки; нормализация: хранить как пустую строку или не включать в payload (на усмотрение реализации, консистентно с фронтом и бекендом).

## 4. Заметки докладчика в python-pptx

- После создания/заполнения слайда обратиться к `slide.notes_slide` (при первом обращении python-pptx создаёт notes slide при необходимости).
- Текст заметок: `notes_slide.notes_text_frame` — возвращает `TextFrame` или `None` (если в образце нет notes placeholder).
- Запись текста: установить текст в `notes_text_frame.paragraphs[0]` или очистить и добавить параграф; при отсутствии `notes_text_frame` — пропустить запись без ошибки.
- Ограничение длины текста перед записью: обрезать до MAX_NOTES_LEN (4000) символов.
