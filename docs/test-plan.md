## 1. Приоритеты покрытия

| Уровень | Приоритет | Что покрывать в первую очередь |
|--------|-----------|--------------------------------|
| **Unit (backend)** | Высокий | Валидация входных данных (slides: пустой массив, >100 слайдов, неверный JSON, лимиты заголовка/пункта/количества пунктов); парсинг образца pptx (извлечение стиля первого слайда); генерация pptx по описанию слайдов (сервис). |
| **Unit (frontend)** | Высокий | Компоненты формы (форма слайдов, добавление/удаление слайда, валидация полей: 0 слайдов, >100 слайдов, размер файла, формат .pptx); состояния: загрузка (спиннер, кнопка недоступна), успех (кнопки «Скачать», «Создать ещё»), ошибка (detail, кнопка «Повторить»). |
| **Integration (API)** | Высокий | POST /api/presentation/generate: успешный сценарий (template + slides → 200, бинарный pptx, Content-Type, размер); ошибки 400 (нет template, нет slides, пустой slides, >100 слайдов, неверный JSON), 413 (файл >20 MB), 422 (невалидные поля). |
| **E2E** | Средний | Полный сценарий: форма → ввод слайдов → загрузка .pptx → «Сгенерировать» → успех → «Скачать»; граничные случаи (пустой ввод, неверный формат файла). Реализовано в виде чек-листа для ручной проверки (см. [e2e-checklist.md](e2e-checklist.md)). |

## 2. Unit-тесты бекенда

- **Расположение:** `backend/tests/`
- **Запуск:** из каталога `backend`: `pytest` или `pytest -v`

**Покрытие:**

- **Парсинг образца pptx** (`test_pptx_service.py`): извлечение стиля первого слайда — тесты с реальным маленьким .pptx, созданным в фикстуре (python-pptx), или с моком.
- **Генерация pptx** (`test_pptx_service.py`): на входе описание слайдов + байты образца; на выходе валидный pptx с нужным числом слайдов.
- **Валидация** (`test_validation.py` или внутри `test_api_presentation.py`): slides — пустой массив, >100 слайдов, неверный JSON, лимиты заголовка/пункта/количества пунктов (модели/схемы и логика `_validate_slides`).

---

## 3. Unit-тесты фронтенда

- **Расположение:** `frontend/src/__tests__/` или `frontend/src/test/`
- **Стек:** Vitest + React Testing Library
- **Запуск:** из каталога `frontend`: `npm run test`

**Покрытие:**

- Форма слайдов: добавление/удаление слайда, валидация (0 слайдов, >100 слайдов, размер файла, формат .pptx).
- Состояния: загрузка (спиннер, кнопка недоступна), успех (кнопки «Скачать», «Создать ещё»), ошибка (отображение `detail`, кнопка «Повторить»).

---

## 4. Интеграционные тесты API

- **Расположение:** `backend/tests/test_api_presentation.py`
- **Инструмент:** pytest + TestClient FastAPI (или httpx)
- **Запуск:** `pytest backend/tests/test_api_presentation.py -v` из корня или `pytest tests/test_api_presentation.py -v` из `backend/`

**Сценарии:**

- Успех: загрузка образца + отправка слайдов → 200, бинарный pptx, `Content-Type`, проверка размера ответа.
- Ошибки: 400 (нет template, нет slides, пустой slides, >100 слайдов, неверный JSON), 413 (файл >20 MB), 422 (невалидные поля).

## 5. E2E

- В проекте не настроены Playwright или Cypress.
- Сценарии описаны в виде чек-листа для ручной проверки: [e2e-checklist.md](e2e-checklist.md).
