## 1. User flow: пошаговые сценарии

Ниже описаны сценарии переключения режимов, ввода/редактирования JSON, загрузки через DnD, сохранения в файл и отображения ошибок. Состояния экрана (Форма, Загрузка, Успех, Ошибка) — по [design-spec.md](../design-spec.md). Источник истины — slides state; JSON в режиме «Через JSON» — производное представление (см. [architecture-json-mode.md](architecture-json-mode.md)).

### 1.1 Переключение «По слайдам» ↔ «Через JSON»

| Шаг | Действие пользователя | Поведение UI | Переход |
| --- | -------------------- | ------------ | ------- |
| 1 | Находится в режиме «По слайдам», форма с карточками слайдов заполнена | Видны блоки «Описание слайдов» и «Образец PPTX» по [design-spec](../design-spec.md). В хедере активна кнопка/вкладка «По слайдам». | — |
| 2 | Нажимает «Через JSON» в хедере | Из текущего slides state выполняется сериализация в JSON (например, `JSON.stringify(slides, null, 2)`). Текстовое поле JSON заполняется этим содержимым. Блок карточек слайдов скрыт; виден блок «Ввод через JSON» (textarea + подсказка формата). Блок «Образец PPTX» и кнопка «Сгенерировать» остаются. Активным в хедере становится «Через JSON». | **Режим «По слайдам»** → **Режим «Через JSON»** |
| 3 | Редактирует JSON в textarea (опционально) | Поле редактируемое; при желании — валидация при вводе (debounce) для ранней обратной связи; при ошибке парсинга — под полем сообщение (см. п. 1.4). Slides state при этом не меняется до применения (переключение или «Сгенерировать»). | Остаётся «Через JSON». |
| 4 | Нажимает «По слайдам» в хедере | Содержимое textarea парсится как JSON. Валидация структуры и лимитов по [architecture-json-mode](architecture-json-mode.md). | |
| 4a | Парсинг и валидация успешны | Данные записываются в slides state. Отображается форма по слайдам с обновлёнными данными. Активна кнопка «По слайдам». Ошибка под полем JSON (если была) скрывается. | **Режим «Через JSON»** → **Режим «По слайдам»** |
| 4b | Ошибка парсинга или валидации | Slides state не обновляется. Под textarea отображается сообщение об ошибке (п. 1.4). Режим остаётся «Через JSON»; пользователь может исправить JSON и снова нажать «По слайдам». | Остаётся «Через JSON» (ошибка). |

Переключение в любой момент возможно; при валидном вводе данные не теряются.

### 1.2 Ввод и редактирование JSON в интерфейсе

| Шаг | Действие пользователя | Поведение UI |
| --- | -------------------- | ------------ |
| 1 | Режим «Через JSON» открыт | Видны: textarea с текущим JSON (из slides state), подсказка формата под/над полем (опционально — пример структуры `[{ "title": "...", "items": ["..."] }]` и лимиты). |
| 2 | Вводит или правит текст в textarea | Поле реагирует на ввод. Опционально (MVP): подсветка синтаксиса. Опционально: debounce-валидация с отображением ошибки под полем. |
| 3 | Нажимает «Сгенерировать» | Перед отправкой: парсинг содержимого textarea → валидация. При успехе — данные из парса записываются в slides state (если ещё не синхронизированы), запрос уходит с этим массивом. При ошибке — сообщение под textarea, запрос не отправляется (состояние **Форма** с ошибкой). |

Формат: массив объектов с полями `title` (строка) и `items` (массив строк), лимиты по [api.md](../api.md) — см. requirements и architecture.

### 1.3 Загрузка из файла .json (DnD и выбор файла)

| Шаг | Действие пользователя | Поведение UI |
| --- | -------------------- | ------------ |
| 1 | Видит зону загрузки .json | Зона с текстом вида «Перетащите файл .json сюда или выберите файл»; стиль по ui-spec (п. 2.3 и раздел 2 ниже). Доступна в обоих режимах («По слайдам» и «Через JSON»). |
| 2 | Перетаскивает файл .json в зону или выбирает файл по клику | Проверка: расширение .json, размер файла (рекомендация: макс. 1–2 MB). При превышении размера — сообщение «Файл слишком большой», парсинг не выполняется. |
| 3a | Файл корректен: чтение и парсинг успешны, валидация пройдена | Данные записываются в slides state. В режиме «Через JSON» textarea обновляется из state (подставляется новый JSON). Краткая обратная связь (успех): например, «Файл загружен» или только смена состояния зоны. |
| 3b | Ошибка парсинга (не JSON, пустой файл) | Сообщение: «Файл не является корректным JSON» (или «Пустой файл» для пустого). Slides не меняются. Сообщение рядом с DnD-зоной или в блоке ошибок. |
| 3c | Валидный JSON, но не прошёл структуру/лимиты | Сообщение с указанием причины (см. п. 1.4). Slides не меняются. |

Обработка только на клиенте; файл на бекенд не отправляется.

### 1.4 Сохранение в файл .json

| Шаг | Действие пользователя | Поведение UI |
| --- | -------------------- | ------------ |
| 1 | Нажимает кнопку «Сохранить в JSON» (или «Экспорт в JSON») | Содержимое файла формируется **только из slides state** (`JSON.stringify(slides, null, 2)`). Скачивание файла (например, `presentation.json`) через Blob/URL. Кнопка размещается в зоне режима «Через JSON» или доступна в обоих режимах — по решению реализации; в дизайне рекомендуется видимость в режиме «Через JSON» и при необходимости в «По слайдам». |

Бекенд не используется.

### 1.5 Отображение ошибок (невалидный JSON, лимиты)

Тексты сообщений — по [requirements-json-mode](requirements-json-mode.md) и [architecture-json-mode](architecture-json-mode.md):

| Ситуация | Текст сообщения (пример) | Размещение |
| --------- | ----------------------- | ---------- |
| Синтаксическая ошибка JSON (поле или при переключении) | «Некорректный JSON» или «Некорректный JSON: &lt;причина&gt;» | Под textarea, связь с полем через aria-describedby (п. 3). |
| Корень не массив / нет полей title/items | «Некорректный формат описания слайдов» | Под textarea или у DnD-зоны (при загрузке файла). |
| Пустой массив слайдов `[]` | «Введите хотя бы один слайд» | Под textarea или у блока слайдов. |
| Слайдов &gt; 100 | «Превышено максимальное количество слайдов (100)» | Под textarea или в блоке ошибок. |
| Длина заголовка &gt; 500 символов | «Превышена длина заголовка слайда (500 символов)» (или из api.md) | Аналогично. |
| Длина пункта &gt; 2000 символов | «Превышена длина пункта (2000 символов)» | Аналогично. |
| Пунктов на слайд &gt; 50 | «Превышено количество пунктов на слайд (50)» | Аналогично. |
| Файл не JSON / пустой файл | «Файл не является корректным JSON»; для пустого — «Пустой файл» или тот же текст | Рядом с DnD-зоной. |
| Файл слишком большой | «Файл слишком большой» | Рядом с DnD-зоной. |

Вид сообщений: по [ui-spec](../ui-spec.md) п. 2.4 (блок с фоном `#fef2f2`, граница, текст ошибки). После исправления ввода/файла сообщения скрываются.

## 2. Размещение и вид элементов

Согласованность с [design-spec.md](../design-spec.md) и [ui-spec.md](../ui-spec.md): те же цвета, типографика, стиль кнопок и сообщений об ошибках.

### 2.1 Хедер: переключатель режимов

- **Размещение:** в верхней части страницы, в одной линии с заголовком приложения («Presentation Maker»). Заголовок слева (или по центру), переключатель справа от заголовка или сразу под заголовком — так, чтобы не ломать общую компоновку (верхняя часть по design-spec п. 2.1).
- **Вид:** две кнопки или два таба: **«По слайдам»** и **«Через JSON»**.
  - **Активный режим:** визуальное выделение — фон акцентного цвета или подчёркивание, текст белый или контрастный; неактивный — вторичный стиль (как вторичная кнопка по ui-spec п. 2.1).
  - Стиль кнопок: скругление 6–8 px, padding по вертикали 10–12 px, по горизонтали 16–24 px; шрифт 1rem, 500–600. Акцент: `#2563eb`, hover `#1d4ed8`; вторичный — прозрачный/светло-серый фон, граница.
- **Состояния:** default, hover, focus (видимая обводка 2 px), active. Порядок фокуса: после заголовка переключатель получает фокус (сначала «По слайдам», затем «Через JSON»).

### 2.2 Зона ввода JSON (режим «Через JSON»)

- **Компонент:** многострочное поле (`textarea`). Стиль по ui-spec п. 2.2: минимальная высота 80–100 px, при необходимости больше (например, 200–300 px) или `resize: vertical`; padding 8–12 px, граница 1 px светло-серая, скругление 6 px; при фокусе — граница в цвет акцента.
- **Подсказка формата:** под полем (или над) текст вторичным цветом, 0.875rem: например, «Формат: массив объектов с полями "title" и "items". Пример: [{ "title": "Заголовок", "items": ["Пункт 1"] }]. Максимум 100 слайдов, заголовок до 500 символов, пункт до 2000, до 50 пунктов на слайд.» — при необходимости сократить до одной строки с ссылкой на справку.
- **Placeholder:** опционально, например «Введите JSON…»; не заменять подсказку по лимитам.
- **Ошибка:** блок сообщения об ошибке сразу под textarea (ui-spec п. 2.4), связь с полем через `aria-describedby` и `aria-invalid="true"` (п. 3).

### 2.3 DnD-зона для .json

- **Размещение:** в режиме «Через JSON» — под textarea (или под подсказкой формата); в режиме «По слайдам» — по решению: тот же блок под блоком «Описание слайдов» или в отдельной секции «Загрузить из JSON».
- **Вид:** область с пунктирной или сплошной границей (светло-серый `#e5e7eb`), фон белый или очень светлый (`#fafafa`), скругление 8 px, padding 24–32 px. Текст по центру: «Перетащите файл .json сюда или нажмите для выбора». Шрифт тела или подсказки (0.875–1rem), вторичный цвет.
- **Состояния:** default; **hover/drag-over** — граница в цвет акцента, лёгкий фон (например, `#eff6ff`); **после успешной загрузки** — кратко «Файл загружен» или имя файла (вторичный цвет); **ошибка** — сообщение под/внутри зоны в стиле ошибки (п. 2.4 ui-spec). Сенсорная цель: вся зона кликабельна (минимум высота ~80 px), плюс отдельная кнопка «Выбрать файл» при желании (тогда минимум 44 px высота кнопки).

### 2.4 Кнопка «Сохранить в JSON»

- **Размещение:** в режиме «Через JSON» — рядом с DnD-зоной или под textarea (например, справа от зоны или под ней). Логически рядом с действиями «ввод/загрузка/сохранение» JSON.
- **Вид:** вторичная кнопка по ui-spec п. 2.1 (фон прозрачный или светло-серый, граница, текст основной или акцентный). Текст: «Сохранить в JSON» или «Экспорт в JSON». Те же отступы и скругление, что у остальных кнопок. Состояния: default, hover, focus, active.

### 2.5 Состояния ошибки и подсказки

- **Ошибки:** те же стили, что в design-spec и ui-spec: блок с фоном `#fef2f2`, граница или border-left 4 px в цвет ошибки (`#dc2626`), текст тёмно-красный. Размещение: под связанным полем (textarea) или у DnD-зоны. Тексты — из п. 1.5.
- **Подсказки:** вторичный цвет (`#666`, `#6b7280`), 0.875rem; не путать с сообщениями об ошибках.

## 3. Доступность (a11y)

По [ui-spec.md](../ui-spec.md) раздел 3.

### 3.1 Переключатель режимов

- Фокус: видимый focus indicator (обводка 2 px в цвет акцента или контрастном цвете, outline-offset 2 px). Порядок Tab: заголовок → «По слайдам» → «Через JSON» → далее контент формы (textarea или первый интерактивный элемент блока слайдов).
- Семантика: кнопки с понятным текстом; при использовании табов — `role="tablist"`, `role="tab"`, `role="tabpanel"`, `aria-selected`, `aria-controls` для связи с панелью контента. Активная вкладка: `aria-selected="true"`.

### 3.2 Поле JSON (textarea)

- Связь с ошибкой: при отображении ошибки валидации — `aria-invalid="true"` на textarea; `aria-describedby` указывает на id контейнера с сообщением об ошибке, чтобы скринридер зачитал его после поля.
- Подпись: видимый `<label>` для текстового поля (например, «Описание слайдов в формате JSON») или `aria-label` на textarea.
- Сообщение об ошибке: контейнер с `role="alert"` или `aria-live="assertive"`, чтобы объявлялось сразу при появлении.

### 3.3 DnD-зона

- Доступность с клавиатуры: зона должна получать фокус и активироваться по Enter/Space (открытие диалога выбора файла или input type="file"). Либо отдельная кнопка «Выбрать файл» с явной сенсорной целью ≥ 44 px.
- Описание: `aria-label="Загрузить описание слайдов из файла JSON"` или подпись рядом. При ошибке загрузки — сообщение с `role="alert"` рядом или внутри зоны и логическая связь (например, `aria-describedby` у зоны на id сообщения).

### 3.4 Кнопка «Сохранить в JSON»

- Видимый focus indicator. `aria-label` при необходимости уточнения: «Сохранить описание слайдов в файл JSON».

### 3.5 Общее

- Контраст: все тексты и границы фокуса по WCAG 2.1 Level AA (текст 4.5:1, крупный 3:1).
- Клавиатурная навигация: все элементы доступны через Tab/Enter/Space; при переключении режима фокус переносится на ключевой элемент нового режима (например, в textarea при переходе в «Через JSON»).

## 4. Адаптивность

По [ui-spec.md](../ui-spec.md) раздел 4.

### 4.1 Брейкпоинты

- Мобильный (до 576 px): переключатель в хедере — в одну строку (кнопки компактнее при необходимости) или вертикальный стек; textarea и DnD-зона на полную ширину контентной области с отступами 16 px от краёв.
- Планшет (576–992 px) и десктоп (≥ 992 px): центральная колонка макс. 720–800 px; переключатель в хедере в одну линию с заголовком.

### 4.2 Отступы и сенсорные цели

- Отступы кратны 8 px (8, 16, 24, 32). Расстояние между блоками «Ввод через JSON», DnD-зона, кнопка «Сохранить в JSON» — 16–24 px.
- Кнопки переключателя и кнопка «Сохранить в JSON»: минимальная высота сенсорной цели 44 px (padding или min-height). DnD-зона на мобильных — не менее 80 px по высоте для удобного нажатия.

### 4.3 Ширина контента

- Максимальная ширина контента (textarea, DnD, форма слайдов) — 720–800 px, центрирование. На малых экранах — без горизонтальной прокрутки при viewport от 320 px.
