### 1.2. Граничные случаи

| Случай | Описание | Покрытие |
|--------|----------|----------|
| Пустой JSON / пустой массив | `[]` → ошибка «Введите описание хотя бы одного слайда» | ✅ `parseAndValidateSlides` |
| Битый JSON | Синтаксическая ошибка → «Некорректный JSON» | ✅ `parseAndValidateSlides` |
| Не массив | Объект, число, строка в корне → «Некорректный формат описания слайдов» | ✅ `parseAndValidateSlides` |
| Отсутствие title/items или неверный тип | Не строка title, не массив items, элемент items не строка | ✅ `parseAndValidateSlides` |
| Слайдов > 100 | TOO_MANY_SLIDES | ✅ `parseAndValidateSlides` |
| Заголовок > 500 символов | TITLE_TOO_LONG(n) | ✅ `parseAndValidateSlides` |
| Пункт > 2000 символов | ITEM_TOO_LONG(slide, item) | ✅ `parseAndValidateSlides` |
| Пунктов на слайд > 50 | TOO_MANY_ITEMS(n) | ✅ `parseAndValidateSlides` |
| Файл .json слишком большой | MAX_JSON_FILE_SIZE (2 MB) — логика в App (processJsonFile) | Константа и сообщение в constants; интеграционно — при необходимости ручная проверка |
| Пустая или нестрока на входе parseAndValidateSlides | Пустая строка, пробелы → INVALID_JSON | ✅ `parseAndValidateSlides` |

## 2. Unit-тесты

### 2.1. `frontend/src/utils/slidesValidation.js` — `parseAndValidateSlides`

**Файл тестов:** `frontend/src/__tests__/slidesValidation.test.js`  
**Раннер:** Vitest.

- **Валидный ввод:** один слайд, несколько слайдов, пустой items, ровно MAX_SLIDES/MAX_TITLE_LEN/MAX_ITEM_LEN/MAX_ITEMS_PER_SLIDE.
- **Невалидный JSON:** битая строка, незакрытая скобка, лишняя запятая, неверные кавычки.
- **Не строка / пустая строка:** `""`, `"   "`, строка `"42"` (после парса не массив).
- **Не массив:** объект, число, строка, null в корне.
- **Пустой массив:** `[]` → NO_SLIDES.
- **Формат слайда:** отсутствует title/items, title не строка, items не массив, элемент items не строка; элемент массива null/число/строка.
- **Лимиты:** слайдов >100, title >500, item >2000, items >50 на слайд; проверка номера слайда/пункта в сообщении.

**Итого:** 37 тестов.

### 2.2. `frontend/src/App.jsx` — режим JSON

**Файл тестов:** `frontend/src/__tests__/App.test.jsx` (блок «App – режим JSON»).

- Переключатель режимов: кнопки «По слайдам» и «Через JSON» в хедере.
- Переключение в «Через JSON» подставляет в поле JSON из текущих слайдов.
- Переключение в «По слайдам» с валидным JSON обновляет слайды (заголовок и пункты).
- Переключение в «По слайдам» с невалидным JSON показывает ошибку в role="alert".
- Кнопка «Сохранить в JSON» вызывает `URL.createObjectURL` с Blob типа `application/json`, размер > 0, затем `revokeObjectURL`.
- В режиме JSON отображаются textarea, подсказка, кнопка сохранения и зона загрузки JSON.

**Итого:** 6 тестов (режим JSON).

## 3. Интеграционные / E2E сценарии

E2E-фреймворк в проекте не настроен. Рекомендуемые сценарии для ручной или будущей E2E-автоматизации:

1. **Полный цикл:** переключение в «Через JSON» → ввод валидного JSON → «Сохранить в JSON» → проверка скачанного файла → переключение в «По слайдам» → проверка слайдов → выбор PPTX → «Сгенерировать» → проверка ответа.
2. **Загрузка файла:** выбор .json через «Загрузить из JSON» (click) → проверка обновления слайдов и при необходимости текста в режиме JSON; то же для drag-and-drop.
3. **Граничные случаи в UI:** загрузка не-.json, файл > 2 MB, пустой файл, битый JSON — проверка сообщений об ошибке и отсутствия подмены данных.
4. **Генерация из JSON:** ввод валидного JSON → выбор PPTX → «Сгенерировать» → проверка, что в запрос уходит корректный массив слайдов (при наличии инструмента перехвата запросов).
