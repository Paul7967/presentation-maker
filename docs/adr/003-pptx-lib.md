# ADR 003: Библиотека для работы с PPTX — python-pptx

**Статус:** Принято  
**Дата:** 2025-03-06  
**Контекст:** Этап 2.2 — работа с pptx (извлечение стиля из образца, генерация презентации).

---

## Решение

Использовать библиотеку **python-pptx** (пакет `python-pptx` в PyPI) для:
- чтения загруженного образца PPTX и извлечения стиля/структуры первого слайда;
- создания новой презентации и записи слайдов с применением извлечённого стиля.

---

## Контекст и обоснование

**Рассмотренные варианты:**

1. **python-pptx** — чисто Python, работа с Office Open XML (pptx как ZIP + XML). Широко используется, активная поддержка, достаточная документация. Позволяет читать слайды, фигуры, параграфы, шрифты, отступы и создавать новые презентации программно.

2. **python-pptx + низкоуровневая работа с OOXML** — для тонкой настройки можно при необходимости разбирать XML внутри пакета; по умолчанию достаточно высокоуровневого API python-pptx.

3. **Другие библиотеки (например, только парсинг через openpyxl для xlsx)** — не подходят, т.к. pptx имеет другую структуру (presentation.xml, slide layouts, theme и т.д.). Специализированная библиотека для pptx предпочтительна.

4. **Вызов внешних приложений (LibreOffice, MS Office)** — усложняет деплой и окружение, создаёт зависимость от внешнего софта; для серверной генерации по описанию не требуется.

**Выбор python-pptx:**

- Нативная работа с форматом PPTX в Python без внешних зависимостей.
- Возможность и читать, и писать презентации; извлечение стилей из существующих слайдов и применение к новым реализуемо в рамках API.
- Подходит для MVP (один образец — первый слайд; генерация по списку заголовок + пункты).

**Ограничения:**

- Сложные макеты и темы могут потребовать доработки логики извлечения (например, мастер-слайды, несколько placeholder’ов). Для MVP достаточно воспроизведения стиля текста и базовой структуры первого слайда.
- При необходимости расширения (несколько образцов, выбор слайда-образца) текущее решение допускает эволюцию без смены библиотеки.

## Последствия

- В backend добавляется зависимость `python-pptx` в requirements.txt.
- Реализация сервиса генерации: модуль чтения образца (извлечение стиля первого слайда) и модуль записи новой презентации по описанию слайдов с применением этого стиля.
- Обработка ошибок парсинга (повреждённый файл, пустая презентация) — см. PRD и api.md (сообщения пользователю и коды ошибок).
