# User Stories и критерии приёмки: Presentation Maker

**Версия:** 1.0  
**Дата:** 2025-03-06  
**Связь с документом:** [PRD/ТЗ](prd.md)

---

## Приоритизация

- **MVP** — минимально жизнеспособный продукт: ввод описания слайдов, загрузка одного образца PPTX, генерация и скачивание PPTX, базовая валидация и сообщения об ошибках.
- **Пост-MVP** — последующие итерации: расширение формата ввода, выбор слайда-образца, улучшение UX и производительности.

---

## MVP

### US-1. Ввод текстового описания слайдов

**Как** автор презентации,  
**я хочу** ввести заголовок и пункты для каждого слайда в структурированной форме,  
**чтобы** система могла сгенерировать презентацию с нужным содержимым.

**Приоритет:** MVP

**Критерии приёмки:**

- [ ] Пользователь может ввести минимум один слайд с заголовком и списком пунктов (пункты — массив строк, один уровень для MVP).
- [ ] Пользователь может добавить несколько слайдов (порядок слайдов сохраняется).
- [ ] Пустые пункты допускаются; пустой заголовок слайда допускается (слайд всё равно создаётся).
- [ ] При отправке формы без ни одного слайда или с пустым описанием показывается ошибка валидации.

**Given** пользователь на странице создания презентации,  
**When** он вводит один и более слайдов с заголовками и пунктами и нажимает «Сгенерировать»,  
**Then** описание передаётся на бекенд в согласованном формате (например, JSON).

---

### US-2. Загрузка образца слайда (PPTX)

**Как** автор презентации,  
**я хочу** загрузить файл презентации в формате PPTX в качестве образца,  
**чтобы** новая презентация имела такой же стиль и структуру слайда.

**Приоритет:** MVP

**Критерии приёмки:**

- [ ] Пользователь может выбрать файл с расширением .pptx и загрузить его через контрол выбора файла.
- [ ] Допускается только формат PPTX; при выборе файла другого формата показывается ошибка (или файл не принимается).
- [ ] При превышении максимального размера файла показывается сообщение об ошибке с указанием лимита.
- [ ] После выбора файла отображается имя загруженного файла (и при необходимости размер).

**Given** пользователь на странице создания презентации,  
**When** он выбирает файл .pptx в пределах лимита размера,  
**Then** файл принимается и отправляется на бекенд при запросе генерации.

**Given** пользователь выбирает файл не в формате PPTX или размером выше лимита,  
**When** он пытается отправить форму или загрузить файл,  
**Then** отображается понятное сообщение об ошибке и запрос на генерацию не выполняется (или файл не принимается).

---

### US-3. Запуск генерации презентации

**Как** автор презентации,  
**я хочу** нажать одну кнопку (например, «Сгенерировать»), чтобы запустить создание презентации после ввода описания и загрузки образца,  
**чтобы** не выполнять лишних действий и быстро получить результат.

**Приоритет:** MVP

**Критерии приёмки:**

- [ ] Кнопка запуска генерации активна только при наличии и описания слайдов, и загруженного образца PPTX.
- [ ] При нажатии отправляется запрос на бекенд с описанием слайдов и файлом образца.
- [ ] Во время обработки отображается индикатор загрузки/ожидания (например, спиннер или «Генерация…»).
- [ ] Пользователь не может повторно отправить запрос до завершения текущего (или явно отменить — опционально для MVP).

**Given** заполнены описание слайдов и загружен образец PPTX,  
**When** пользователь нажимает «Сгенерировать»,  
**Then** отправляется запрос на генерацию и показывается состояние «загрузка».

---

### US-4. Скачивание готовой презентации (PPTX)

**Как** автор презентации,  
**я хочу** после успешной генерации скачать готовый файл PPTX,  
**чтобы** использовать презентацию в своих целях.

**Приоритет:** MVP

**Критерии приёмки:**

- [ ] После успешной генерации пользователю предлагается скачать файл (кнопка «Скачать» или автоматическое скачивание).
- [ ] Скачиваемый файл имеет расширение .pptx и корректно открывается в приложениях, поддерживающих PPTX.
- [ ] Количество слайдов в файле равно количеству слайдов в описании.
- [ ] Содержимое слайдов (заголовки и пункты) соответствует введённому описанию.
- [ ] Внешний вид слайдов (шрифты, отступы, расположение) соответствует образцу (первому слайду загруженного PPTX).

**Given** генерация завершилась успешно,  
**When** пользователь нажимает «Скачать» (или получает файл автоматически),  
**Then** файл PPTX сохраняется на устройство и открывается без ошибок в стандартном приложении.

---

### US-5. Обработка ошибок валидации и сервера

**Как** автор презентации,  
**я хочу** видеть понятные сообщения об ошибках при неверном вводе или сбое генерации,  
**чтобы** понять, что исправить, и повторить попытку.

**Приоритет:** MVP

**Критерии приёмки:**

- [ ] При пустом описании слайдов отображается сообщение типа «Введите описание хотя бы одного слайда».
- [ ] При отсутствии загруженного образца отображается сообщение «Загрузите образец в формате PPTX».
- [ ] При неверном формате файла (не PPTX) — сообщение «Поддерживается только формат PPTX».
- [ ] При превышении размера файла — сообщение «Превышен максимальный размер файла» (с указанием лимита).
- [ ] При превышении максимального количества слайдов — сообщение с указанием лимита.
- [ ] При повреждённом или нечитаемом PPTX — сообщение «Файл образца повреждён или не может быть прочитан».
- [ ] При сбое генерации на сервере — сообщение «Не удалось сгенерировать презентацию» (или аналог).
- [ ] После показа ошибки пользователь может изменить данные и повторить запрос.

**Given** пользователь отправил запрос с невалидными данными или произошла ошибка сервера,  
**When** ответ возвращается с кодом ошибки,  
**Then** в интерфейсе отображается соответствующее текстовое сообщение без технического жаргона (по возможности).

---

### US-6. Ограничение количества слайдов

**Как** система,  
**я хочу** ограничить максимальное количество слайдов в одном запросе,  
**чтобы** обеспечить стабильную работу и предсказуемое время ответа.

**Приоритет:** MVP

**Критерии приёмки:**

- [ ] Задано максимальное количество слайдов (значение из архитектуры, например 100 или 200).
- [ ] При превышении лимита запрос не выполняется и возвращается ошибка валидации с сообщением о лимите.
- [ ] В интерфейсе при необходимости отображается подсказка о максимальном количестве слайдов.

**Given** пользователь ввёл количество слайдов больше установленного лимита,  
**When** он нажимает «Сгенерировать»,  
**Then** запрос не отправляется или возвращается ошибка с указанием лимита.

## Пост-MVP

### US-7. Многоуровневые списки в пунктах слайда

**Как** автор презентации,  
**я хочу** задавать пункты с подпунктами (несколько уровней вложенности),  
**чтобы** отразить иерархию в содержании слайда.

**Приоритет:** Пост-MVP

**Критерии приёмки:**

- [ ] В формате описания поддерживается вложенность пунктов (например, массив объектов с полем «текст» и «дети» или аналогичная структура).
- [ ] В сгенерированном PPTX подпункты отображаются с соответствующими отступами/нумерацией по образцу.

---

### US-8. Выбор слайда-образца из загруженного файла

**Как** автор презентации,  
**я хочу** выбрать, какой слайд из загруженного PPTX использовать как образец (если в файле несколько слайдов),  
**чтобы** применить нужный макет и стиль.

**Приоритет:** Пост-MVP

**Критерии приёмки:**

- [ ] После загрузки PPTX с несколькими слайдами пользователь может выбрать номер слайда-образца (например, из выпадающего списка).
- [ ] Генерация использует выбранный слайд для извлечения стиля и структуры.
- [ ] По умолчанию используется первый слайд (как в MVP).

---

### US-9. Превью образца перед генерацией

**Как** автор презентации,  
**я хочу** увидеть превью загруженного образца (например, миниатюру первого слайда),  
**чтобы** убедиться, что загружен правильный файл.

**Приоритет:** Пост-MVP

**Критерии приёмки:**

- [ ] После загрузки PPTX отображается превью первого (или выбранного) слайда образца.
- [ ] Превью может быть статичным изображением или упрощённой визуализацией; реализация уточняется на этапе дизайна.

---

### US-10. Отмена генерации

**Как** автор презентации,  
**я хочу** отменить запрос на генерацию, пока он выполняется,  
**чтобы** не ждать завершения при ошибочном запуске или смене решения.

**Приоритет:** Пост-MVP

**Критерии приёмки:**

- [ ] Во время ожидания ответа отображается кнопка «Отменить».
- [ ] При нажатии «Отменить» запрос прерывается (если технически возможно), индикатор загрузки скрывается, форма снова доступна для редактирования.
