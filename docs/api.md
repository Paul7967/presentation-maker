# API: Presentation Maker

**Версия:** 1.0  
**Дата:** 2025-03-06  
**Базовый путь (пример):** `/api`  
**Связь:** [Архитектура](architecture.md), [ADR 002](adr/002-api.md), [PRD](prd.md)

---

## 1. Обзор

API предназначен для генерации презентации PPTX по загруженному образцу и текстовому описанию слайдов. В MVP один эндпоинт: **POST /api/presentation/generate**.

---

## 2. Лимиты и таймауты

| Параметр | Значение | Примечание |
|----------|----------|------------|
| Максимальный размер тела запроса (файл образца + метаданные) | **20 MB** | Ограничить в конфигурации сервера и при валидации |
| Максимальное количество слайдов в описании | **100** | От 1 до 100 включительно |
| Минимальное количество слайдов | **1** | Пустой массив слайдов не допускается |
| Максимальная длина заголовка одного слайда | **500 символов** | Рекомендация для предотвращения переполнения и злоупотреблений |
| Максимальная длина одного пункта | **2000 символов** | Аналогично |
| Максимальное количество пунктов на один слайд | **50** | Для стабильности и предсказуемого времени ответа |
| Таймаут выполнения запроса генерации | **120 секунд** | При превышении возвращать 504 Gateway Timeout |

## 3. Эндпоинт: генерация презентации

### POST /api/presentation/generate

Принимает образец PPTX и описание слайдов; возвращает сгенерированный файл PPTX или JSON с ошибкой.

#### Запрос

- **Content-Type:** `multipart/form-data`
- **Поля:**
  - `template` (обязательно) — файл с расширением `.pptx`, MIME-тип `application/vnd.openxmlformats-officedocument.presentationml.presentation` или `application/octet-stream`. Размер в пределах лимита (20 MB).
  - `slides` (обязательно) — строка в кодировке UTF-8, содержащая JSON-массив описания слайдов.

**Формат поля `slides` (JSON):**

```json
[
  {
    "title": "Заголовок первого слайда",
    "items": ["Пункт 1", "Пункт 2", "Пункт 3"]
  },
  {
    "title": "Второй слайд",
    "items": []
  }
]
```

- **title** — строка, заголовок слайда (может быть пустой).
- **items** — массив строк, пункты списка (для MVP один уровень). Пустой массив допускается.

**Пример минимального валидного запроса:** один слайд с заголовком и одним пунктом.

#### Успешный ответ

- **Код:** 200 OK
- **Content-Type:** `application/vnd.openxmlformats-officedocument.presentationml.presentation`
- **Content-Disposition:** `attachment; filename="presentation.pptx"`
- **Тело:** бинарное содержимое файла PPTX.

#### Ошибки (тело — JSON)

Формат ответа с ошибкой (стандарт FastAPI/HTTP):

```json
{
  "detail": "Текстовое сообщение об ошибке"
}
```

или для ошибок валидации (несколько полей):

```json
{
  "detail": [
    { "loc": ["body", "slides"], "msg": "...", "type": "..." }
  ]
}
```

Для единообразия на фронтенде достаточно обрабатывать `detail`: если это строка — показать её; если массив — сформировать сообщение из полей `msg` или объединить их.

| Код | Ситуация | Пример сообщения (detail) |
|-----|----------|---------------------------|
| **400** | Нет поля `template` или файл не передан | «Загрузите образец в формате PPTX». |
| **400** | Нет поля `slides` или пустая строка | «Передайте описание слайдов в поле slides». |
| **400** | Неверный JSON в `slides` | «Некорректный формат описания слайдов». |
| **400** | В описании 0 слайдов или пустой массив | «Введите описание хотя бы одного слайда». |
| **400** | Количество слайдов &gt; 100 | «Превышено максимальное количество слайдов (100)». |
| **400** | Нарушение лимитов (длина заголовка/пункта, кол-во пунктов на слайд) | «Превышен лимит: &lt;конкретика&gt;». |
| **400** | Файл не в формате PPTX (MIME или сигнатура) | «Поддерживается только формат PPTX». |
| **413** | Размер файла превышает 20 MB | «Превышен максимальный размер файла (20 MB)». |
| **422** | Unprocessable Entity — ошибка валидации полей (например, Pydantic) | Структура `detail` с полями `loc`, `msg`, `type`. |
| **500** | Файл образца повреждён или не может быть прочитан | «Файл образца повреждён или не может быть прочитан». |
| **500** | В образце нет ни одного слайда | «В файле образца не найден ни один слайд». |
| **500** | Ошибка при извлечении стиля или генерации PPTX | «Не удалось сгенерировать презентацию». |
| **504** | Таймаут запроса (например, &gt; 120 с) | «Превышено время ожидания. Попробуйте уменьшить количество слайдов или размер образца». |

Все сообщения должны быть понятны пользователю (без технического жаргона, где возможно).

## 4. Валидация на бекенде

- **Файл template:** проверка наличия; размер до 20 MB; MIME-тип или сигнатура (ZIP/OOXML). Расширение `.pptx` можно проверять дополнительно, но не полагаться только на него.
- **Поле slides:** наличие; валидный JSON; тип — массив; длина массива от 1 до 100; для каждого элемента — наличие полей `title` (строка) и `items` (массив строк); проверка лимитов длины и количества пунктов.
- После валидации — попытка парсинга образца и генерации; при ошибке парсинга/генерации — 500 с соответствующим сообщением.

---

## 5. CORS (для разработки и деплоя)

Для доступа с фронтенда с другого origin (например, localhost:3000 при бекенде на localhost:8000) необходимо настроить CORS на бекенде: разрешить метод POST и заголовки, необходимые для multipart (в т.ч. при необходимости кастомные). В продакшене ограничить `allow_origins` списком доверенных доменов.
