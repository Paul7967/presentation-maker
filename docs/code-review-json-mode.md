# Code Review: режим JSON (Этап 6, dev_plan_2.md)

**Область:** изменения по вводу/загрузке/сохранению JSON и переключению режимов.  
**Проверенные файлы:** `frontend/src/constants.js`, `frontend/src/utils/slidesValidation.js`, `frontend/src/App.jsx`, `frontend/src/App.css`.

---

## 1. Безопасность

### 1.1 Парсинг JSON без eval — **OK**

- В коде **нет использования `eval`** (поиск по `frontend` не нашёл вхождений).
- В `slidesValidation.js` парсинг выполняется **только через `JSON.parse(jsonString)`** (стр. 18–19). В JSDoc явно указано: «Uses only JSON.parse() (no eval)».

### 1.2 Ограничение размера загружаемого .json файла — **OK**

- В `constants.js` задано ограничение: `MAX_JSON_FILE_SIZE = 2 * 1024 * 1024` (2 MB).
- В `App.jsx` в `processJsonFile` проверка размера выполняется **до** чтения файла:
  - сначала проверка расширения (`.json`);
  - затем `if (fileObj.size > MAX_JSON_FILE_SIZE)` с установкой ошибки и `return`;
  - только после этого создаётся `FileReader` и вызывается `reader.readAsText()`.
- Таким образом, слишком большой файл не читается и не парсится — критерий выполнен.

## 3. Сообщения об ошибках

- Сообщения заданы в `MESSAGES` и используются единообразно.
- **Рекомендация:** для `JSON_FILE_TOO_BIG` («Файл слишком большой.») можно указать лимит в сообщении, например: «Файл слишком большой (максимум 2 MB).» — так пользователю проще понять ограничение.
- `JSON_FILE_INVALID` используется и при неподходящем расширении, и при невалидном содержимом/ошибке чтения. Для расширения можно завести отдельное сообщение (например, «Поддерживается только файл с расширением .json»), чтобы различать сценарии — по желанию, не обязательно.

## 4. Доступность (a11y)

### 4.1 Переключатель режимов — **OK**

- Контейнер: `role="tablist"` и `aria-label="Режим ввода"`.
- Кнопки: `role="tab"`, `aria-selected={inputMode === "bySlides"}` / `"byJson"`.
- Состояние выбранной вкладки доступно для скринридеров.

### 4.2 Режим «Через JSON» — **OK**

- У `textarea` есть `id="json-slides"`, связанный с `label` (класс `visually-hidden`).
- Используются `aria-invalid={!!jsonError}` и `aria-describedby` на подсказку или блок ошибки (`id="json-hint"` / `id="json-error"`).
- Кнопка «Сохранить в JSON» имеет `aria-label="Сохранить описание слайдов в файл JSON"`.

### 4.3 Зона DnD для JSON — **OK**

- Контейнер: `role="button"`, `tabIndex={0}`, `aria-label="Загрузить описание слайдов из файла JSON"`, при ошибке — `aria-describedby="json-file-error"`.
- Обработка клавиш: `onKeyDown` для `Enter` и `Space` открывает выбор файла — поведение соответствует кнопке.
- Скрытый `<input type="file">` с `aria-hidden="true"`, чтобы не дублировать фокус для скринридеров.

### 4.4 Остальное — **OK**

- Блоки ошибок с `role="alert"`.
- Состояние загрузки: `aria-busy="true"`, `aria-live="polite"`, спиннер с `aria-hidden="true"`.

Критерии по aria/role соблюдены.
